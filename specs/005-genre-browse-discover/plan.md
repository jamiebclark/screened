# Implementation Plan: Genre-Linked Browse & Discover Page

**Branch**: `005-genre-browse-discover` | **Date**: 2026-05-17 | **Spec**: [spec.md](spec.md)

## Summary

Add a `/browse` page backed by TMDB's discover API, with genre pill filtering, media type toggle, and optional user-specific filters (seen/unseen, in-library, friends' library). Genre badges on movie detail pages, TV detail pages, and list item modals become clickable links to the Browse page. The redundant "Home" and "Search" top-nav links on detail pages are removed.

## Technical Context

**Language/Version**: TypeScript / Node 20 (Next.js 15 App Router)
**Primary Dependencies**: Next.js RSC, Prisma, TMDB REST API, Tailwind v4, Radix UI
**Storage**: PostgreSQL via Prisma (read-only for this feature — no schema changes)
**Testing**: Vitest (unit), Playwright (E2E)
**Target Platform**: Web (server-rendered, self-hosted via Docker Compose)
**Project Type**: Web application
**Performance Goals**: Browse page initial load < 2s; filter changes via URL navigation
**Constraints**: TMDB rate limits (3-retry with backoff already handled); post-filtering means pages may show fewer than 20 results when user filters are active
**Scale/Scope**: Single new route + changes to 4 existing files

## Implementation Phases

### Phase 1 — TMDB lib additions (`src/lib/tmdb.ts`)

Add four functions using the existing `tmdbFetch` helper:

```
getMovieGenres()  → TmdbGenre[]           revalidate: 86400
getTvGenres()     → TmdbGenre[]           revalidate: 86400
discoverMovies(genreId?, page?)  → TmdbSearchResponse   revalidate: 3600
discoverTv(genreId?, page?)      → TmdbSearchResponse   revalidate: 3600
```

`TmdbGenre = { id: number; name: string }` — already defined inline on existing models; extract as a named export.

`discoverMovies`/`discoverTv` params: `with_genres=<id>` (omit if not provided), `sort_by=popularity.desc`, `page=<n>`.

**Commit**: `feat(tmdb): add genre list and discover API wrappers`

---

### Phase 2 — Browse page RSC (`src/app/(app)/browse/page.tsx`)

Server component. Search params: `type`, `genre`, `genreName`, `filter`, `page`.

**Resolution logic (server-side)**:

1. Parse `type` → default `"movie"`
2. If `type !== "all"`: fetch genre list (`getMovieGenres` or `getTvGenres`), find active genre by `genre` param (ID) or `genreName` param (name lookup)
3. Fetch discover results: `discoverMovies` / `discoverTv` with active genre ID and page, or `getTrending("all","week")` when `type === "all"`
4. If session exists: call `getUserTmdbMediaStateByRef` for watch status overlay
5. If `filter=friends` and session: call `listFriendUserIds` then query `UserMediaStatus` to find which results are tracked by friends
6. If `filter=library` and session: query `UserMediaStatus` for user's tracked tmdbIds, filter results
7. If `filter=seen/unseen` and session: filter results against user's `WATCHED` status items

**Page structure** (content hierarchy per constitution):

```
<h1>Browse</h1>
<BrowseFilters ... />        ← "use client" component (genre pills, type toggle, filter row)
<div grid>                   ← MediaCard per result, with status overlay
<PaginationControls />       ← previous/next page links
```

**Commit**: `feat(browse): add /browse discover page`

### Phase 3 — BrowseFilters client component (`src/app/(app)/browse/browse-filters.tsx`)

`"use client"`. Receives: `genres`, `activeGenreId`, `type`, `filter`, `isLoggedIn` as props (passed from RSC — no client-side fetching).

On interaction: build new URLSearchParams and call `router.push()`. No `router.refresh()` needed (navigating to new URL re-renders RSC naturally).

**Genre pills**: horizontal scroll row, active pill highlighted. "All Genres" pill clears genre param.
**Type toggle**: Movies | TV Shows | All — buttons that update `?type=`.
**Filter row** (logged-in only): Seen | Not Seen | In My Library | Friends' Library — toggle buttons that update `?filter=`.

**Commit**: included in the browse page commit above (same logical unit)

### Phase 6 — Genre links in list item modal (`src/app/(app)/lists/[slug]/list-item-modal.tsx`, lines ~99-103)

Modal genres are `String[]`. Replace the comma-joined text with individual badge links:

```tsx
// before
{
  mediaItem.genres.length > 0 && (
    <p className="text-xs text-muted-foreground mt-1">
      {mediaItem.genres.join(", ")}
    </p>
  );
}

// after
{
  mediaItem.genres.length > 0 && (
    <div className="flex flex-wrap gap-1 mt-1">
      {mediaItem.genres.map((g) => (
        <Link
          key={g}
          href={`/browse?genreName=${encodeURIComponent(g)}&type=${type === "MOVIE" ? "movie" : "tv"}`}
          onClick={onClose}
        >
          <Badge
            variant="secondary"
            className="text-xs hover:bg-secondary/80 transition-colors cursor-pointer"
          >
            {g}
          </Badge>
        </Link>
      ))}
    </div>
  );
}
```
