# Implementation Plan: Upcoming from Watchlist

**Branch**: `004-upcoming-watchlist` | **Date**: 2026-05-05 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/004-upcoming-watchlist/spec.md`

## Summary

Dedicated `/upcoming` page surfacing the authenticated user's watchlist items that have a known release date — grouped into "Coming Soon" (future releases) and "Just Released" (released in the past 30 days, still on watchlist). Requires bumping `CURRENT_ENRICHMENT_VERSION` from 1 to 2 so older `MediaItem` records without `releaseDate` are backfilled on next access. No schema migrations needed — `MediaItem.releaseDate` already exists.

## Phase 0: Research

**Resolved unknowns** (from codebase exploration):

| Question                                              | Finding                                                                                                                       |
| ----------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| Is `releaseDate` already on `MediaItem`?              | Yes — `DateTime?`, migration `20260503175016`. Populated at creation time.                                                    |
| Do older items have `releaseDate = null`?             | Yes — enrichment does not currently set it; items pre-dating the migration will have `null`.                                  |
| How to backfill?                                      | Bump `CURRENT_ENRICHMENT_VERSION` 1→2; add `releaseDate` update inside `enrichAndEmbed` when current value is null.           |
| Does the watchlist page already query upcoming items? | Yes — a "Releasing soon" section exists (limited to 12, future only). Keep it as a teaser; add "See all" link to `/upcoming`. |
| How to detect "unwatched" for Just Released?          | `UserMediaStatus.status = 'WATCHLIST'` is sufficient — users who have watched a title change its status to WATCHED.           |
| Where do nav links live?                              | `navLinks` array in `src/components/nav.tsx` (client component, no server data needed).                                       |
| Appropriate Lucide icon for upcoming?                 | `CalendarDays` — already imported in the watchlist page for the same concept.                                                 |
