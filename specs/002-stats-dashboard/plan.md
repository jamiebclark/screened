# Implementation Plan: Stats Dashboard

**Branch**: `002-stats-dashboard` | **Date**: 2026-05-06 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/002-stats-dashboard/spec.md`

## Summary

Dedicated `/stats` page showing per-user watch statistics: total hours/titles/average rating summary, genre and decade breakdowns (CSS percentage bars), ratings histogram, and top-10 directors and actors. All data is aggregated server-side in a single lib function from existing `WatchEntry`, `UserMediaStatus`, and `MediaItem` tables. No schema changes, no new API routes, no new dependencies.

## Technical Context

**Language/Version**: TypeScript 5, Node 20
**Primary Dependencies**: Next.js 15 App Router (RSC), Prisma ORM, Tailwind v4, Lucide icons
**Storage**: PostgreSQL via Prisma (read-only for this feature)
**Testing**: Playwright (E2E)
**Target Platform**: Web (self-hosted, Docker)
**Performance Goals**: Page renders in < 3 s for 500-item watch history
**Constraints**: Auth required; no schema migrations; no new charting library; no `"use client"` needed
**Scale/Scope**: Per-user query; watch history typically < 500 items

## Phase 1: Design

### Data Model

Two Prisma queries run in parallel; all aggregation in JavaScript. See [data-model.md](./data-model.md).

**Q1** — All `MediaItem` records the user has watched (via `WatchEntry`):

```ts
prisma.mediaItem.findMany({
  where: { watchEntries: { some: { userId } } },
  select: { id, runtime, genres, year, cast, directors, director, mediaType },
});
```

**Q2** — All non-null ratings for the user:

```ts
prisma.userMediaStatus.findMany({
  where: { userId, rating: { not: null } },
  select: { rating },
});
```

**`StatsData` return type** (full shape in data-model.md):

- `summary` — totalTitles, totalHoursWatched, averageRating
- `genreBreakdown[]` — genre, count, percentage
- `decadeBreakdown[]` — decade, count, percentage
- `ratingsHistogram[]` — rating, count, percentage
- `topDirectors[]` — name, count (top 10)
- `topActors[]` — name, count (top 10)

### UI Layout

```
/stats
├── Header: BarChart2 icon badge + "Stats" h1 + subtitle
├── Summary cards row (3 cards): Total Hours | Titles Watched | Avg Rating
├── Genre Breakdown (h3 section): horizontal bars, ranked
├── Decade Breakdown (h3 section): horizontal bars, chronological
├── Ratings Histogram (h3 section): vertical bars, 0.5–5.0
├── Top Directors (h3 section): ranked list, count badge
├── Top Actors (h3 section): ranked list, count badge
└── Empty state (when no watch history): centered, link to /history
```

Section headings follow the standard: `<h3 className="text-base font-semibold">`.
Sections with no data render an inline "No data yet" note rather than hiding the section.

### No API Contracts

This feature is a pure RSC with no new API routes. No contracts file needed.

## Implementation Notes

### `stats-queries.ts` aggregation pattern

```ts
// Genre breakdown: flatten String[] arrays, count, sort desc
const genreCounts = new Map<string, number>();
for (const item of watchedItems) {
  for (const g of item.genres) {
    genreCounts.set(g, (genreCounts.get(g) ?? 0) + 1);
  }
}

// Decade breakdown: derive from year
const decade = item.year ? `${Math.floor(item.year / 10) * 10}s` : null;

// Directors: prefer directors[] array, fallback to director string
const allDirectors = [
  ...item.directors,
  ...(item.directors.length === 0 && item.director ? [item.director] : []),
];
```

### Bar chart rendering (no library)

```tsx
{
  genreBreakdown.map(({ genre, count, percentage }) => (
    <div key={genre} className="flex items-center gap-3">
      <span className="w-28 text-sm text-right truncate">{genre}</span>
      <div className="flex-1 h-4 bg-muted rounded-full overflow-hidden">
        <div
          className="h-full bg-primary rounded-full"
          style={{ width: `${percentage}%` }}
        />
      </div>
      <span className="text-sm tabular-nums text-muted-foreground w-6 text-right">
        {count}
      </span>
    </div>
  ));
}
```

### Empty state

When `summary.totalTitles === 0`, render a full-page empty state with a link to `/history`, following the pattern in `watchlist/page.tsx`.
